# Data

This directory contains all data used in the ML workflow.
Data files are excluded from version control via `.gitignore`
and must be added manually before running any notebooks.

## Directory structure

data/
├── raw/          # original immutable data — never modified
├── processed/    # preprocessed data, generated by notebook 2
├── interim/      # engineered features, generated by notebook 4
└── external/     # third party reference data, lookup tables,


## Getting started

### Adding your dataset
1. Place your raw data file(s) in `data/raw/`
2. Update `config.yaml` with your target column and columns to drop:
```yaml
target: "your_target_column"
drop_columns:
  - "your_id_column"
```
3. The workflow supports any single CSV file — no other changes needed

## When to use external/
Use the `external/` folder for supplementary data that enriches
your main dataset but comes from a separate source. Examples:
- Country or region lookup tables
- Industry classification codes
- Economic or demographic reference data
- Data retrieved from external APIs

## Notes
- `data/raw/` is the only folder you need to populate manually
- `data/processed/` and `data/interim/` are generated automatically
  by running the notebooks in order
- Add any external reference files to `data/external/` manually
- Never modify files in `data/raw/` — treat them as immutable
- Never commit data files to GitHub — they are intentionally
  excluded via `.gitignore`
